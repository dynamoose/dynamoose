    Plugin~Plugin(model, func, options, registerPlugin)

    This is a helper to bind new plugins and the events they trigger to your model. This helper is consumed by the Model, and not directly when creating a Plugin.

    Kind: inner method of Plugin

    Param Type Description
    model class the passed in model class
    func function the user provided plugin function
    options object the user provided options
    registerPlugin function the bound model plugin method

    plugin.emit(type, stage, obj)

    Sets up the model for all built listeners

    Kind: instance method of Plugin

    Param Type Description
    type string the type of model change occuring
    stage string the stage, pre or post, of the type being emitted
    obj object the Plugin object to configure for listeners

    Plugin~on(type, stage, func)

    This allows for the ability for plugin to add listeners on certain events emmited from Dynamoose. This is the function that gets called from your plugin.on() hook.

    This example will reject a put if they attempt to modify a key you don't want modified.

    plugin.on('model:put', 'put:called', (pluginEvent) => {
   const {event, model} = pluginEvent;
   const ItemToTransform = event.item.Item;
   const mutatingKeys = Object.keys(putQuery);
   const staticOnlyKeys = 'id'
   const result = {}
   if (mutatingKeys.includes(staticOnlyKeys)) {
     result.reject = 'This key must not be included, we manage it.';
   } else {
     result.resolve = 'Good to go.'
   }
   return Promise.resolve(result)
 }
)

    Kind: inner method of Plugin

    Param Type Description
    type mixed if a string, sets what type to listen to, a function sets listeners on all event stages and types
    stage mixed if a string, sets the stage to listen to, a function sets listeners on all stage for the preceding type
    func function the function to call with the correct payloads at the defined type and stage
